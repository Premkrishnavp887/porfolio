# AGENTS.md — astro-byte-zen

## What this is

Client-side static SPA built with React 19 + TanStack Router. **Not Astro** despite the repo name. Zero SSR — all rendering happens on the client. Content is data-driven from JSON files in `src/data/`.

## Commands

```bash
bun install            # deps (bun is primary; bun.lock committed)
bun run dev            # vite dev server with HMR
bun run build          # vite build → dist/client/
bun run preview        # vite preview of production build
bun eslint src/        # lint (no lint script; run eslint directly)
bun prettier --write src/  # format (no format script; run prettier directly)
```

No test runner configured.

## Architecture

- **Entry**: `index.html` → `src/main.tsx` → `getRouter()` + `createRoot()`.
- **Routing**: TanStack Router, file-based routes in `src/routes/`. Currently `/` and `/projects/$slug`. `__root.tsx` is the app shell — keep `<Outlet />` intact.
- **Data**: `src/data/*.json` → parsed at import by Zod schemas in `src/schemas/` → re-exported from `src/data/index.ts`. Edit JSON, not components.
- **Components**: shadcn/ui in `src/components/ui/`, portfolio sections in `src/components/portfolio/`, project detail in `src/components/project/`.
- **Config**: Centralized constants in `src/config/` — Framer Motion springs/durations in `animationConfig.ts`, theme keys in `themeConfig.ts`.

## Stack

| Layer           | Choice                                                         |
| --------------- | -------------------------------------------------------------- |
| Framework       | React 19 + TanStack Router                                    |
| Build           | Vite 8 + `@vitejs/plugin-react` + `@tailwindcss/vite`          |
| Styling         | Tailwind CSS v4 + `tw-animate-css`                             |

| Lint            | ESLint (typescript-eslint) + Prettier                           |

## Conventions

- **Path alias**: `@/` → `src/`.
- **CSS**: Tailwind v4 with `@theme inline` + `@utility` in `src/styles.css`. Dark-first; `.light` class on `<html>` switches palette.
- **Theme**: `dark`/`light`/`system` persisted to localStorage key `astro-portfolio-theme`. Inline flash-prevention script in `index.html`.
- **Route tree**: `src/routeTree.gen.ts` is auto-generated by TanStack Router — don't edit manually. No codegen script is configured; adding routes requires either activating the Vite plugin (`@tanstack/router-plugin/vite`) or running `tsr generate` manually.
- **JSON data**: Adding a field requires editing both the JSON file and its corresponding Zod schema.
- **Dead SSR files**: `src/server.ts` and `src/start.ts` are unused remnants of a prior TanStack Start setup.
